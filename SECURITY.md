# Security Policy

## Supported versions

This repository is an actively-developed toolkit, not a released product. Only the current `main` branch receives security fixes. Older commits and tags are historical snapshots.

## Scope

**In scope:**
- Source code under `lib/`, `scripts/`, `canaries/`, and any agent specs distributed in this repo
- Credential handling in `lib/credentials/` and the OS keychain integration
- Multi-provider orchestration in `lib/orchestrator/` (API proxy, adapter code, finding pipeline)
- Build and CI artifacts generated by this repo

**Out of scope:**
- Upstream AI provider behavior (Anthropic, Google, xAI, OpenAI, or any third-party CLI vendor) — file those with the upstream
- Third-party npm dependencies — track supply-chain issues via `make qa-sca`; security fixes there happen in the upstream package
- Target repositories analyzed by this toolkit
- Forks or mirrors of this repository

## Reporting a vulnerability

Preferred channels, in order:

1. **GitHub private vulnerability reporting** — use the "Report a vulnerability" button on the Security tab of this repository.
2. **Email:** **salesinfo@sparfuchs-pro.com**  ·  Subject prefix: **[security]**

Please include:
- Repo state (commit SHA) and affected configuration
- Reproduction steps or a proof-of-concept
- Impact assessment
- Your name or handle if you would like credit

Do not open public issues, discussions, or social-media posts about the vulnerability during the disclosure window below.

## Disclosure window

- **Acknowledgement:** within 3 business days of receipt.
- **Initial triage and severity classification:** within 7 business days.
- **Coordinated disclosure:** up to 90 days from the date of the report before public disclosure. Extendable by mutual agreement for complex or infrastructure-level fixes.
