# Contributing

Thanks for your interest in contributing to `sparfuchs-qa`.

## Ground rules

- By submitting a contribution, you agree that your work is licensed under the [Apache License 2.0](./LICENSE).
- All contributions are made via pull request. Direct pushes to `main` are not permitted.
- All commits must be signed (`git commit -S`) and/or include a `Signed-off-by` trailer (`git commit -s`).

## Workflow

1. Fork the repository or create a branch (for collaborators).
2. Make your changes with clear, focused commits.
3. Open a pull request against `main`.
4. A CODEOWNER review and passing status checks are required before merge.
5. PRs are merged via squash-merge; keep your PR title descriptive — it becomes the commit message.

## Code of Conduct

Please be respectful. Harassment, discrimination, and abusive behavior are not tolerated.

## Reporting security issues

See [SECURITY.md](./SECURITY.md). Do not file public issues for vulnerabilities.
